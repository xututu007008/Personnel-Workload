# 机关摸排报告排版与生成规则

本规则用于后续生成机关工作量摸排相关 Markdown 与 PDF 报告，目标是在更新数据的同时尽量保持参考文件的章节结构、表格样式和 PDF 阅读效果一致。

## 一、生成原则

1. 不覆盖历史参考文件，新报告按日期生成新文件，例如 `报告名称-YYYY-MM-DD.md` 与 `报告名称-YYYY-MM-DD.pdf`。
2. 先生成 `机关员工主管评价20题重算报告`，后续所有涉及 `按20题计算等级`、`规则重算评级` 的报告均以该报告口径为准。
3. 生成报告时应参考同目录下旧版 MD/PDF 的章节顺序，不得只生成摘要版。
4. PDF 格式应与参考 PDF 接近，重点保持表格表头着色、行高舒展、列宽合理、图片正常渲染。
5. 所有专项报告应由一个主脚本统一生成，避免多个脚本口径不一致。

## 二、报告生成顺序

建议按以下顺序生成：

1. `机关员工主管评价20题重算报告`
2. `机关员工ERP使用情况分析`
3. `机关员工工作负荷报告`
4. `机关员工证书客观维度分析`
5. `机关工作量摸排报告`
6. `机关工作量摸排结论分析`
7. `重点问题反馈汇总`
8. `重点问题反馈结论分析`
9. `机关工作量摸排综合报告`

## 三、PDF表格基础样式

PDF 中所有表格应采用统一样式：

1. 表头背景色使用浅蓝色，建议 `#D9EAF7`。
2. 表格边框使用浅灰色，建议 `#BFBFBF`。
3. 单元格垂直对齐使用顶部对齐。
4. 单元格上下内边距应适当放大，避免行高过挤；建议上下内边距不低于 `5`。
5. 表格正文允许自动换行，不应把同一个人员或同一个控件的多条意见拆成多个单元格。
6. 同一主管针对同一员工的多条评价，应合并到一个单元格中，用 `<br>` 或分号分隔。
7. 同一员工针对同一问题控件的多条意见，也应合并到一个单元格中。

## 四、PDF列宽规则

列宽应按表格语义设置，不应简单平均分配。

### 1. 两列表格

短统计表可等宽：

- `等级 / 有效反馈人数`
- `角色特征 / 人数`
- `负荷层级 / 人数`
- `差异档位 / 人数`
- `口径 / 涉及人数`
- `口径 / 发起流程总数`
- `选项 / 人数`

部门或人员明细类两列表格应左窄右宽：

- `部门 / 差异人员`
- `部门 / 人员名单`
- `部门 / 主要反馈方向`

建议比例：

- 左列 `30-36mm`
- 右列占剩余宽度

### 2. 三列表格

`档位 / 人数 / 部门及人员` 或 `等级 / 人数 / 部门及人员` 必须使用短列加宽明细列：

- `档位`：约 `18mm`
- `人数`：约 `18mm`
- `部门及人员`：剩余宽度

`等级 / 工作状态倾向 / 人数` 应左右窄、中间宽：

- `等级`：约 `24mm`
- `人数`：约 `24mm`
- `工作状态倾向`：剩余宽度

`ERP信号 / 人员 / 复核含义` 应保证“复核含义”有足够宽度：

- `ERP信号`：约 `30mm`
- `人员`：约页面宽度的 `40%-45%`
- `复核含义`：剩余宽度

### 3. 多列表格

4列以上表格应优先保证长文本列宽度，例如：

- `想法明细`
- `主管评价反馈明细`
- `异常选项`
- `主观想法明细`
- `部门及人员`

人数、等级、分数、日期等短字段应压缩为窄列。

## 五、图片规则

1. Markdown 中的图片语法应在 PDF 中真实渲染，不应显示为“图片引用”文字。
2. `机关工作量摸排综合报告` 的 `基础分级明细` 前应包含 `各部门等级热力矩阵对比` 图片。
3. 矩阵图应对比至少两个维度：
   - 主管原始评价等级分布
   - 规则重算评级分布
4. 矩阵图应使用当前数据重新生成，不直接复用旧图。
5. 图片插入 PDF 时应按页面宽度等比缩放，最大高度不得超过可打印区域。

## 六、重点报告结构要求

### 1. 机关员工ERP使用情况分析

必须参考 `ERP使用情况.xlsx` 中：

- `近三年`
- `职能部门人员台账`

必须明确：

- 机关统计总人数
- 有 ERP 痕迹人数
- 无 ERP 痕迹人数
- 部门分布
- 角色特征分布
- 按人员评价等级观察 ERP 使用情况
- 重点人员表现
- 无ERP痕迹情况
- 异常信号
- 结论判断

### 2. 机关员工证书客观维度分析

必须在主管评价20题重算之后生成。

必须包含：

- 按等级部门汇总
- 年终总结未匹配名单
- 人力资源证书复核明细
- 无任何证书人员按主管评价等级分布
- 工作满2年仍无任何证书人员名单
- 工作未满2年且无任何证书人员名单

人力资源复核资料中显示已有证书的人员，必须从无证书人员中剔除。

### 3. 机关员工工作负荷报告

`五、差异情况` 下必须包含：

- 员工自评与主管认定差异较大样本
- 自动负荷与主管认定差异较大样本

### 4. 机关工作量摸排报告

必须包含：

- 各部门工作摸排填报情况
- 未填报工作摸排人员名单
- 各部门员工评价完成情况
- 工作负荷与岗位状态
- 本维度结论
- ABCD 等级分布情况
- 自动评级总体分布
- 自动评级与主管主观评级一致性
- 自动评级与主管主观评级不一致人员名单
- 20题规则重算评级与主管主观评级一致性
- 20题规则重算评级与主管主观评级不一致人员名单
- 各部门主管主观评级结构
- 主管评级与薪酬区间一致性
- 岗位必要性、离职影响与评级反差
- 各部门等级结构是否符合常态分布
- 取低档后的异常复核名单
- 低负荷信号与等级交叉复核范围
- 特殊人群与分层管理规则
- 综合结论与建议

### 5. 重点问题反馈汇总

必须注意：

1. `妈妈岗有意向人员名单` 仅纳入明确有意向人员，例如选项 `A.有强烈意向` 和 `B.有意向，但是不迫切，需考虑`。
2. 明确选择无意向的人员不得放入有意向名单。
3. `家庭照护与岗位优化倾向统计` 应包含各问题选项统计，包括 `未填写`。
4. `岗位优化或调整机会下的工作状态倾向` 应增加等级维度。
5. 同一人员同一问题下的多条意见应合并在一个单元格。

### 6. 重点问题反馈结论分析

应保留旧稿中的归纳性承接内容，例如：

- `典型部门与反馈方向如下：`
- `其中较为典型的建议包括：`
- `具体较集中的反馈有：`
- `因此，主管侧信息更适合用于：`
- `因此，这一块现阶段更适合判断为：`

不应只输出章节标题和人数统计，应保留分析性文字与典型方向表。

### 7. 机关工作量摸排综合报告

不得生成简单索引摘要，应包含完整正文结构：

- 总体业务流程
- 填写情况
- 主管评价基础分级结果
- 规则重算校验结果
- 基础分级明细
- 各部门等级热力矩阵对比图
- 规则重算评级与主管评价差异人员
- 局部复核和调整建议
- 证书维度信号
- 人力资源复核后的收紧说明
- 工作负荷维度信号
- ERP 使用情况信号
- 岗位价值和关键题目异常信号
- 未满1年人员保护原则
- 综合结论
- 附件建议

## 七、数据口径规则

1. 机关总人数应参考 `ERP使用情况.xlsx` 中的 `职能部门人员台账`，并按旧稿口径补入或剔除特殊人员。
2. `南京永利重工制造有限公司` 人员按旧稿口径补入。
3. 主管评价等级相关分析必须优先使用 `规则重算评级`。
4. `主管主观评级` 可作为管理判断保留，但不得替代规则重算评级。
5. 年终总结证书、主管评价表证书和人力资源复核证书应合并去重。
6. 若数据缺失，应在报告中说明缺口，而不是删除参考章节。

## 八、脚本实现建议

1. 所有报告由一个主脚本统一生成。
2. Markdown 与 PDF 同步生成。
3. PDF 生成器应支持：
   - Markdown 标题
   - Markdown 表格
   - Markdown 图片
   - 表格重复表头
   - 表头着色
   - 表格按列语义设置列宽
4. 生成 PDF 前应先生成依赖图片，例如等级热力矩阵。
5. 运行后应校验：
   - 文件是否生成
   - PDF 页数是否合理
   - PDF 是否包含图片
   - 关键章节是否存在

## 九、交付检查清单

交付前至少检查：

- `机关员工主管评价20题重算报告` 是否最先生成
- 后续报告中的 `按20题计算等级` 是否与重算报告一致
- 旧稿已有章节是否缺失
- 表头是否为浅蓝色
- 行高是否舒展
- 短列是否被压缩
- 长文本列是否足够宽
- 图片是否在 PDF 中真实显示
- 同一人员同一问题的多条反馈是否合并
- 新文件是否未覆盖旧文件
