# 机关工作量摸排结论分析规则

## 一、规则定位

本文件用于指导生成、检查和修改 `机关工作量摸排结论分析`。该报告是 `机关工作量摸排报告` 的结论化版本，面向管理判断和后续复核安排。

## 二、必须读取的输入

- `机关工作量摸排报告`
- `机关员工主管评价20题重算报告`
- `机关员工工作负荷报告`
- `机关员工ERP使用情况分析`
- `机关员工证书客观维度分析`

## 三、结论写法

1. 所有结论必须来自已生成分报告。
2. 当前范围为单部门时，不得写成“机关整体普遍存在”。
3. 当前范围为部分部门时，应写“当前样本范围内”。
4. 若人数为 0，应写“当前未发现”，不得继续列虚假问题。
5. 建议分为 `建议复核`、`建议关注`、`可作为后续管理观察` 三类，不直接替代最终定档。

## 四、必须包含的章节

1. 总体结论
2. 评价结论
3. 后续建议

如用户要求细化，应扩展为：

- 填报与评价完成情况
- 等级结构判断
- 工作负荷判断
- 岗位必要性和评级反差
- 证书和客观维度补充
- ERP 使用辅助判断
- 风险人员或重点复核人员
- 管理建议

## 五、验收清单

- 与主报告人数一致。
- 与综合报告方向一致。
- 不出现和 0 人统计矛盾的固定套话。
- 单部门报告用单部门描述。

## 六、逐项结论标准

| 结论项 | 判断来源 | 写法标准 |
| --- | --- | --- |
| 填报完整性 | 摸排填报率、评价完成率 | 说明样本是否足以支撑阶段性判断。 |
| 等级结构 | 主管主观评级、20题重算评级 | 说明高低档集中情况，不用公司等级字段。 |
| 工作负荷 | 工作负荷报告 | 区分低负荷、满负荷、差异样本。 |
| 岗位必要性 | 主管评价关键题 | 只作为复核信号，不直接形成定档。 |
| 证书客观维度 | 证书报告 | 区分无证书、HR复核后仍无证书、荣誉补充。 |
| ERP 使用 | ERP报告 | 作为承担度辅助信号，不替代评价。 |
| 重点反馈 | 反馈汇总和结论 | 区分员工想法、主管反馈、妈妈岗意向。 |

## 七、可执行生成流程

1. 先确认 `机关工作量摸排报告`、主管评价重算报告、工作负荷报告、ERP报告、证书报告均已生成；如某分报告缺失，应在本报告对应章节写明缺失，不得虚构结论。
2. 读取主报告中的填报率、评价完成率、未填报人员、未评价人员、等级结构和异常复核名单。
3. 读取工作负荷报告中的低负荷、满负荷和差异较大样本。
4. 读取证书报告中的复核后无证书人员、工作满2年仍无证书人员、荣誉补充人员。
5. 读取 ERP 报告中的无痕迹人员、流程量靠前人员、异常信号人员。
6. 将上述信号按人员聚合，形成 `建议复核`、`建议关注`、`后续观察` 三类，不直接写最终定档结论。
7. 按当前范围输出，单部门写“该部门”，部分部门写“当前样本范围”，全量写“机关范围”。

## 八、建议分类规则

| 分类 | 纳入条件 | 写法 |
| --- | --- | --- |
| 建议复核 | 多个维度同时出现低负荷、无证书、等级偏高、岗位必要性偏弱等信号 | 写明触发信号和需复核事项。 |
| 建议关注 | 单一维度出现明显异常，但证据不足以形成复核 | 写明继续观察口径。 |
| 后续观察 | 新员工、未满1年、样本不完整、主管评价缺失 | 写明保护或补样建议。 |

## 九、0 人和缺失样本写法

1. 某类风险人员为 0 时，写 `当前未发现该类人员`，不得继续列典型问题。
2. 主管评价样本为 0 时，评价结论只能写样本缺失和后续补评建议，不得分析等级结构。
3. 工作摸排样本不足时，必须提示结论仅能作为阶段性观察。
4. 单部门样本少于 3 人时，避免使用“结构明显失衡”等强判断。

## 十、输出结构模板

1. `总体结论`：样本完整性、主要可用数据、当前判断边界。
2. `评价结论`：主管评价和20题重算等级结构；无主管评价时写补评建议。
3. `工作负荷结论`：低负荷、满负荷、差异样本。
4. `客观维度结论`：证书、ERP、荣誉等辅助信号。
5. `后续建议`：按复核、关注、观察三类输出。

## 十一、生成后复核方法

1. 结论中出现的每个人都能在分报告中找到来源。
2. 不出现与统计人数矛盾的句子，例如 `0人` 后继续列主要意见。
3. 建议语气保持辅助复核，不直接替代最终定档。
