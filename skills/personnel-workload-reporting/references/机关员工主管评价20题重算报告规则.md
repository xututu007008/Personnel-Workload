# 机关员工主管评价20题重算报告规则

## 一、规则定位

本文件用于指导生成、检查和修改 `机关员工主管评价20题重算报告`。凡用户要求调整主管评价 20 题重算、等级口径、逐题分值、匹配异常、主管主观评级对比、后续报告等级同步时，必须优先读取本规则。

本报告是全部机关摸排报告的前置报告。后续报告中出现的 `按20题计算等级`、`20题重算评级`、`综合重算评级`、`基础分级`，均应以本报告的重算结果为准。

## 二、必须读取的输入

1. `部门成员工作主管评定*.xlsx`
   - 读取员工姓名、部门、主管主观评级、表内自动评级、20 题原始选项文本、表内题项分值、岗位必要性、薪资区间等字段。
   - 如果同一员工存在多条主管评价记录，不得合并为一条平均记录，应逐条列示并分别计算。

2. `C1 机关员工主管评价赋分规则-2026-05-23.md`
   - 这是本报告的逐题赋分主规则。
   - 修改代码或报告时，必须以该文件中的题目、选项、视同处理、分值、档位区间、岗位必要性扣分为准。

3. `机关员工主管评价赋分规则*.md`
   - 若当前文件夹中存在同名新版本，应优先读取最新版本，再与 `C1` 文件核对差异。

4. `职能部门工作量摸排表*.xlsx`
   - 用于辅助识别当前部门范围，确保单部门、部分部门、全部部门输出口径一致。

## 三、范围识别

1. 若当前文件夹只放入一个部门的数据，输出前缀为 `部门名-`。
2. 若当前文件夹放入多个但非全部部门的数据，输出前缀为 `部分部门-`，报告内要列明涉及部门。
3. 若当前文件夹为全部机关范围，输出前缀为 `全部部门-` 或无部门前缀，按现有脚本风格执行。
4. 人员姓名应统一去空格、全半角差异和明显换行差异；部门归属优先以当前主管评价表和工作摸排表中的有效部门为准。

## 四、计算规则

### 1. 20 题基础分

1. 原始主管评价表共包含 `20` 道编号题。
2. 不得直接采用表内 `题1分值` 至 `题20分值` 作为最终结果。
3. 应读取每题原始选项文本，按 `C1 机关员工主管评价赋分规则-2026-05-23.md` 第三章逐题匹配分值。
4. 20 题得分相加形成 `纯20题重算总分`。
5. 匹配分值采用 `0.5` 分刻度。

### 2. 匹配方式

每题匹配结果必须区分以下类型：

| 类型 | 含义 | 报告展示 |
| --- | --- | --- |
| 精确题 | 原始选项文本与规则中的题目和选项完全或规范化后匹配 | 计入 `精确X题` |
| 字母题 | 原始文本无法完全匹配，但可按 `A/B/C/D/E/F` 选项字母定位 | 计入 `字母X题` |
| 未匹配题 | 既不能精确匹配，也不能按字母可靠定位 | 计入 `未匹配X题`，并应进入异常复核 |

当用户询问某员工“字母1题是哪一题”时，应能定位到具体题号、题目、原始选项、匹配分值和匹配原因。

### 3. 岗位必要性修正

`该成员所在岗位是否有存在的必要性` 为独立观察项，按当前已确认口径纳入综合重算分：

| 选项 | 修正分 |
| --- | ---: |
| A.需要存在，不可去除 | 0 |
| B.存不存在均可 | -1 |
| C.建议与其他岗位合并 | -3 |
| D.直接去掉，完全没有存在的必要性 | -5 |

输出中必须同时保留：

- `纯20题重算总分`
- `岗位必要性修正分`
- `综合重算总分`
- `综合重算评级`

### 4. 档位区间

| 分数区间 | 等级 |
| --- | --- |
| 95分及以上 | S |
| 85-94.5分 | A |
| 75-84.5分 | B |
| 60-74.5分 | C |
| 50-59.5分 | D |
| 40-49.5分 | E |
| 40分以下 | F |

边界分值必须严格按闭区间处理，例如 `84.5` 为 B，`85` 为 A。

## 五、报告必须包含的章节

### 一、测算口径

必须说明：

- 本报告读取主管评价表原始选项文本重新赋分。
- 不直接采用表内题项分值。
- 岗位必要性已作为修正项纳入综合重算总分。
- 后续报告中的 20 题等级均使用本报告结果。

### 二、整体结果

至少包含：

- 纳入主管评价记录数
- 涉及员工数
- 涉及部门数
- 表内自动评级分布
- 主管主观评级分布
- 20题规则重算评级分布

### 三、样本校核结果

至少包含：

- 表内总分、纯20题重算总分、岗位必要性修正分、综合重算总分的统计。
- 表内自动评级与重算评级不一致人数。
- 主管主观评级与重算评级不一致人数。
- 精确匹配、字母匹配、未匹配题的统计。

### 四、人员明细

明细表必须逐人逐记录列示，不得只给汇总结论。

## 六、明细表字段顺序

人员明细建议列顺序如下：

| 列名 | 说明 |
| --- | --- |
| 部门 | 当前归属部门 |
| 姓名 | 员工姓名 |
| 表内20题总分 | 原表题项分值合计，仅用于对照 |
| 纯20题重算总分 | 按规则重算的 20 题得分 |
| 岗位必要性修正分 | 岗位必要性扣分或 0 |
| 综合重算总分 | 纯20题重算总分 + 岗位必要性修正分 |
| 表内基础分级 | 原表自动生成评级 |
| 综合重算评级 | 本报告最终基础等级 |
| 主管评价等级 | 主管主观评级 |
| 规则匹配情况 | 精确X题/字母X题/未匹配X题 |

如需增加逐题明细，可按 `题号、题目、原始选项、匹配选项、分值、匹配方式` 展开。

## 七、不得使用的口径

1. 不得用 `主管评级对应公司等级` 参与等级分析。
2. 不得用薪酬区间反推重算等级。
3. 不得把主管主观评级当作规则重算评级。
4. 不得把表内自动评级当作重算评级。

## 八、PDF 排版规则

1. 表头统一浅蓝底色。
2. 明细表属于宽表，PDF 中应压缩等级、姓名、分值列，保留规则匹配情况列可读宽度。
3. 行高不得过挤，中文长文本上下需有留白。
4. 宽表左右页边距不得过小。
5. 同一人员长文本不得被挤成竖排。

## 九、修改代码时的定位

常见修改点：

- 读取规则文件：查找 `parse_supervisor_rules`、`rule`、`score`、`question`。
- 逐题匹配：查找 `match`、`letter`、`unmatched`。
- 岗位必要性修正：查找 `岗位必要性`、`necessity`、`修正分`。
- 报告生成：查找 `build_supervisor_report`。
- 后续报告同步等级：查找 `规则重算评级`、`按20题计算等级`、`综合重算评级`。

## 十、验收清单

- 随机抽查 1 至 3 名员工，逐题分值与 `C1` 规则一致。
- 用户指定人员时，能定位到异常题号。
- `纯20题重算总分 + 岗位必要性修正分 = 综合重算总分`。
- 综合重算评级与档位区间一致。
- 后续所有报告等级与本报告一致。

## 十二、四大维度标准

| 维度 | 题号 | 合计分 | 分析含义 |
| --- | --- | ---: | --- |
| 日常交付与靠谱程度 | 1、2、3、5、9、11 | 30.5 | 判断员工是否高效、稳定、省心、能闭环，属于当前执行可靠性的核心维度。 |
| 工作态度与团队协作 | 7、8、10、12、15、17 | 29 | 判断主动优化、跨部门协同、团队角色、边缘任务态度、共享意识和改进态度。 |
| 工作负荷和抗压能力 | 6、13、14 | 12.5 | 判断负荷状态、纪律规范和高压承接能力，不单独决定高档位。 |
| 学习成长与综合价值 | 4、16、18、19、20 | 28 | 判断业务理解、学习速度、同岗水平、成长上限和离职影响。 |

## 十三、逐题异常复核规则

1. 若某题出现 `未匹配`，报告中必须保留原始选项文本，不能直接给 0 分后沉默处理。
2. 若某题使用 `字母匹配`，必须优先确认同题号下该字母只有一个可用选项；如同字母存在多个历史选项，应进入人工复核。
3. 若旧版问卷串入项已在 `C1` 中写明视同处理，则按 `C1` 执行，不计为未匹配。
4. 若旧版问卷串入项未在 `C1` 中写明，不能自行猜测，应列入未匹配题并在报告中提示。
5. 用户指定人员复核时，输出顺序应为：姓名、部门、题号、题目、原始选项、匹配方式、匹配分值、是否影响等级。

## 十四、可执行生成流程

1. 确认当前目录是否存在 `部门成员工作主管评定*.xlsx`。若不存在，仍生成报告，整体结果写 `纳入测算评价记录：0 条`，总分统计写 `暂无主管评价样本`，人员明细表只保留表头。
2. 读取主管评价表时使用 `Sheet1` 且按现有表头行读取；若表头位置变化，应先定位 `成员姓名`、`成员部门`、`主管主观评级`、`自动生成评级`、20题题干列和 `该成员所在岗位是否有存在的必要性`。
3. 按当前范围过滤人员：单部门取当前部门，部分部门取当前文件夹内摸排表或主管表中出现的部门，全部部门取机关范围。范围过滤必须在重算前完成。
4. 对每条主管评价记录执行：清洗姓名和部门、提取20题原始选项、逐题匹配规则、计算纯20题总分、计算岗位必要性修正分、得到综合重算总分和等级。
5. 输出整体分布时按 `S、A、B、C、D、E、F` 顺序展示，即使某等级为 0 也应在分布表中保留列。
6. 输出明细时按部门顺序、姓名、评价时间排序；同一员工多名主管评价分别列示，不得均值合并。
7. 生成完成后将本报告结果作为后续所有报告的等级来源，后续报告不得自行重新解释等级。

## 十五、字段映射与缺失处理

| 目标字段 | 优先来源 | 缺失处理 |
| --- | --- | --- |
| 部门 | `成员部门` | 缺失时尝试从工作摸排表按姓名匹配；仍缺失则写空，不参与部门汇总。 |
| 姓名 | `成员姓名` | 缺失记录剔除。 |
| 主管姓名 | `主管姓名(必填)` | 缺失写空。 |
| 评价时间 | `评价时间(必填)` | 缺失写空；排序时放后。 |
| 表内自动评级 | `自动生成评级` | 缺失写空，不得替代为重算评级。 |
| 主管主观评级 | `主管主观评级` | 缺失写空，不纳入主管评级分布。 |
| 主管薪资档次 | `档级` | 仅供综合报告前后对比使用。 |
| 主管定档薪资 | `定档薪酬` | 仅供综合报告前后对比使用。 |
| 岗位必要性 | `该成员所在岗位是否有存在的必要性` | 缺失按 0 修正，但在匹配情况或复核说明中保留。 |

## 十六、空样本和异常样本写法

1. 主管评价记录为 0 时，不得调用最低分、最高分、分位数等统计函数。
2. 主管评价记录为 0 时，报告仍应生成 `测算口径`、`整体结果`、`样本校核结果`、`人员明细` 四章。
3. 未匹配题数量大于 0 时，必须在明细或附表中列出题号；若当前版报告暂不展开逐题明细，应至少在 `规则匹配情况` 中显示未匹配数量。
4. 若同一员工多条记录等级差异较大，不做自动合并，后续综合报告按主管评价记录口径逐条或按最新记录口径展示时必须说明。

## 十七、生成后复核方法

1. 随机抽查一名高分、一名中分、一名低分员工，按 `C1` 规则逐题手工核对。
2. 检查 `综合重算总分 = 纯20题重算总分 + 岗位必要性修正分`。
3. 检查边界分值：95、85、84.5、75、74.5、60、59.5、50、49.5、40 是否落入正确等级。
4. 检查后续报告中的 `按20题计算等级` 与本报告一致；如不一致，必须先修正后续报告，不得改本报告等级迁就后续结果。
