# 机关工作量摸排综合报告规则

## 一、规则定位

本文件用于指导生成、检查和修改 `机关工作量摸排综合报告`。本报告是所有分报告的最终汇总版本，必须在其他报告全部生成后再生成。

## 二、生成顺序

1. `机关员工主管评价20题重算报告`
2. `机关员工ERP使用情况分析`
3. `机关员工工作负荷报告`
4. `机关员工证书客观维度分析`
5. `机关工作量摸排报告`
6. `机关工作量摸排结论分析`
7. `有想法人员及妈妈岗意向名单`
8. `重点问题反馈汇总`
9. `重点问题反馈结论分析`
10. `机关工作量摸排综合报告`

## 三、核心口径

1. 所有等级以 `机关员工主管评价20题重算报告` 为主。
2. 主管评价基础分级章节中，必须先显示 `主管主观评级分布`，再显示 `按主管评价20题赋分规则重算后的等级分布`。
3. `主管评级对应公司等级` 不参与分析。
4. 当前范围为单部门、部分部门或全部部门时，全文范围描述必须一致。
5. 人力资源复核不涉及当前范围人员时，不显示复核说明。
6. 无人员的风险项显示 `无`，不得继续生成具体意见或名单。
7. ERP 维度统一引用 `ERP使用情况.xlsx` 中 `2025年全年` 工作簿，不再引用 `近三年` 工作簿。
8. 当前范围主管评价样本为 0 时，不生成主管主观评级分布、20题规则重算分布、基础分级矩阵和前后对比空表，只保留“当前范围未取得主管评价样本”的说明。

## 四、必须包含的章节

1. 工作背景
2. 摸排范围及完成情况
3. 未填报工作摸排人员名单
4. 未完成主管评价人员名单
5. 主管评价及规则重算情况
6. 基础分级矩阵
7. 等级热力矩阵图
8. 规则重算评级与主管评价前后对比
9. 辅助维度复核情况
10. 证书维度
11. 工作负荷维度
12. ERP 使用情况
13. 未满1年人员保护原则
14. 综合结论
15. 附件材料

## 五、文字风格

综合报告正文应参考正式公文报告或实施意见的表达方式，避免简单的数据罗列口吻。文字风格要求如下：

- 正文标题采用“关于……情况的报告”表达，不在标题中附加日期；日期只保留在输出文件名中，避免 PDF 首页出现单独日期行。
- 第一部分采用“为进一步……，本轮……”句式说明工作背景和目的。
- 正文多使用“从……看”“经……校验”“总体看”“建议……”等公文表达。
- 章节结构按“一、”“（一）”展开，弱化技术化小标题。
- 数据表作为支撑材料保留，表前后使用结论性文字衔接。
- 不在正文中过度使用反引号、代码样式或技术说明式表达。
- PDF 或 Word 版排版可交由 WPS AI 公文排版处理，脚本同时应尽量直接输出符合公文规范的 PDF 字体层级。
- 脚本自带 PDF 输出字体规则：
  - 文件标题：优先使用 `方正小标宋简体`，字号为二号字（约 22pt）；如本机未安装该字体，则优先回退 WPS 字体包中的 `FZSSK.ttf`（方正书宋类）。
  - 大章节标题：优先使用 `黑体`，字号为三号字（约 16pt）；如本机未安装，则回退 WPS 字体包中的 `HYZhongJianHeiJ.ttf` 或 `HYQiHei-55J.ttf`。
  - 正文文本：优先使用 `仿宋_GB2312`，字号为三号字（约 16pt）；如本机未安装，则回退 WPS 字体包中的 `FZFSK.ttf`（方正仿宋_GBK）。
  - 正文小标题：优先使用 `楷体_GB2312`，字号为三号字（约 16pt）；如本机未安装，则回退 WPS 字体包中的 `HYKaiTiJ.ttf`（汉仪楷体简）。
  - 以上精确字体文件可放在当前目录 `fonts/` 下，脚本应优先读取当前目录、`fonts/`、用户字体目录、系统字体目录和 WPS 字体目录。
- PDF 段落启用 CJK 换行规则，并尽量取消中英文之间不必要空格，例如使用 `ERP使用`、`ERP信号`，避免出现 `ERP` 与 `使用` 被拆到两行的断行。

## 六、证书维度

综合报告中的证书维度必须引用证书报告的收紧口径：

- 职称证书
- 资格证书
- 荣誉证书
- ERP 台账优秀情况
- ERP 台账安全先进个人情况
- HR 证书复核
- 年终总结匹配
- 综合报告正文只呈现证书复核结果和明细，不写“有截图作为佐证时采用收紧口径、无截图则维持原口径”等方法性说明。

## 七、基础分级矩阵与前后对比

必须包含：

- 等级分布说明
- 主管主观评级分布表
- 20题重算评级分布表
- 等级热力矩阵图
- 删除原 `基础分级明细` 宽表，不在综合报告正文展示逐题总分、岗位必要性修正分、规则匹配情况等细项。
- `规则重算评级与主管评价前后对比` 表必须展示所有已完成主管评价人员，包括规则重算评级与主管主观评级一致的人员；一致人员的 `变化情况` 写 `评级一致`。
- 前后对比表列为：部门、姓名、原始主管评价级别、主管薪资档次、主管定档薪资、规则重算评级、重算评级薪资中位数、重算分值、变化情况。
- 主管侧薪资档次读取主管评价表中的 `档级` 字段，即 `1/2/3/4/5档`；主管侧薪资读取 `定档薪酬` 字段，不按等级中位数重新映射。
- 规则重算侧薪资仍按 `规则重算评级` 使用当前等级薪资中位数。
- 前后对比表按主管主观评级和主管薪资档次从高到低排序：评级顺序为 `S>A>B>C>D>E>F`，同一评级内按 `5档>4档>3档>2档>1档` 排序，即综合看 `S5` 最高、`F1` 最低。

## 八、PDF 排版

1. 表头浅蓝底色。
2. 基础分级热力矩阵图必须显示，不得空白。
3. `档位/人数/部门及人员` 表中前两列窄，人员列宽。
4. 证书和异常选项长文本列加宽。
5. 宽表左右页边距不得过小。
6. 行高适中，避免表格挤压。

## 九、验收清单

- 分报告已全部生成。
- 综合报告中的人数、名单、等级与分报告一致。
- 主管主观评级统计在前，20题重算评级统计在后。
- 0 人结论没有矛盾表述。
- 单部门或部分部门描述不泛化为全机关。

## 十、逐项引用标准

| 综合报告模块 | 引用来源 | 必须引用的具体项 |
| --- | --- | --- |
| 填写情况 | 工作量摸排报告 | 部门数、员工数、工作摸排完成率、主管评价完成率、未填报名单、未评价名单 |
| 主管评价基础分级 | 主管评价20题重算报告 | 档位区间、主管主观评级分布、20题重算评级分布、等级热力矩阵、前后等级对比 |
| 证书维度信号 | 证书客观维度分析 | 无证书名单、HR复核后名单、荣誉补充、年终总结未匹配 |
| 工作负荷维度 | 工作负荷报告 | 低负荷人员、差异样本、主管不忙但员工自评忙人员 |
| ERP 使用情况 | ERP分析报告 | 无ERP痕迹、流程量靠前、异常信号、结论判断 |
| 未满1年保护 | 工作量摸排报告 | 工作年限和等级交叉人员 |
| 重点反馈 | 重点问题反馈汇总和结论 | 有想法人员、妈妈岗意向、家庭照护、岗位优化倾向 |

## 十一、综合结论生成标准

1. 综合报告中不再单独生成 `岗位价值和关键题目异常信号` 分析段，避免对单题异常过度敏感；岗位必要性、离职影响等内容仅在明细或专项报告中保留。
2. 综合结论固定为四条：未填报/未评价人数，原始主管评价级别及对应薪资中位数，20题规则重算级别及对应薪资中位数，未满1年人员保护建议。
3. 原始主管评价级别使用 `主管主观评级` 统计，薪资中位数按等级映射展示。
4. 20题规则重算级别使用 `规则重算评级` 统计，薪资中位数按等级映射展示。
5. 若不存在未满1年人员，第四条应明确写“未发现工作未满1年且需要适用至少保留D档保护规则的人员”。
6. “未满1年人员保护原则”等辅助复核章节必须按是否存在人员生成适配语言：无人员时只写“本轮未发现……暂不涉及该项规则”，不得继续输出“当前涉及人员主要为：无”“其中：无……”或后续不适配建议。

## 十二、可执行总流程

1. 先运行主管评价20题重算报告。该报告是所有等级字段的唯一主来源。
2. 再运行 ERP、工作负荷、证书、工作量摸排、工作量结论、有想法名单、重点反馈汇总、重点反馈结论。
3. 最后生成综合报告；综合报告不得重新计算分报告中已经确定的名单和等级，只做引用、交叉和公文式归纳。
4. 每生成一份 MD 后同步生成同名 PDF；文件名保留日期，正文标题不带日期。
5. 当前范围识别必须贯穿所有报告：单部门、部分部门、全部部门不能混用称谓。
6. 若主管评价样本为 0，综合报告保留摸排、证书、ERP、反馈等可用维度，但不生成主管评级分布、20题重算分布、矩阵和前后对比。
7. 若工作摸排样本为 0，综合报告重点写缺失说明和后续补样建议，不得生成工作负荷结论。

## 十三、综合报告字段引用优先级

| 综合报告字段 | 优先来源 | 备用/缺失处理 |
| --- | --- | --- |
| 当前范围部门 | 工作摸排表、主管评价表 | 均缺失时使用 ERP 台账或脚本默认范围，并说明。 |
| 当前范围人数 | ERP台账收紧后的有效员工底表 | 单部门/部分部门按输入表人员收紧。 |
| 未填报人员 | 工作量摸排报告 | 工作摸排表缺失时按当前底表全员未填报。 |
| 未评价人员 | 工作量摸排报告/主管评价表 | 主管评价表缺失时按当前底表全员未评价。 |
| 主管主观评级 | 主管评价表 | 缺失则不统计。 |
| 20题重算评级 | 主管评价20题重算报告 | 缺失则写未评级。 |
| 证书信号 | 证书客观维度分析 | HR复核不涉及当前范围时不显示复核说明。 |
| ERP信号 | ERP使用情况分析 | 使用 `2025年全年` 工作簿口径。 |
| 工作负荷信号 | 工作负荷报告 | 无主管评价时不做主管差异比较。 |
| 重点反馈 | 重点问题反馈汇总/结论 | 0人时写未形成集中反馈。 |

## 十四、公文式正文模板

1. 标题：`关于{范围}工作量摸排及人员评价情况的报告`。
2. 工作背景：用一至两段说明工作目的、采集方式、规则赋分、辅助复核。
3. 摸排范围及完成情况：写部门数、员工数、工作摸排完成率、主管评价完成率、未填报/未评价名单。
4. 主管评价及规则重算情况：有样本时先主管主观评级，再20题重算评级，再矩阵和前后对比；无样本时只写样本缺失说明。
5. 辅助维度复核情况：证书、工作负荷、ERP、重点反馈按实际可用数据写；无人员时不展开不适配建议。
6. 综合结论：固定四条，语言要贴合当前真实数据。
7. 附件材料：列出本次生成的分报告 MD 链接。

## 十五、PDF 公文排版细则

1. 文件标题：方正小标宋简体二号；缺失时用 WPS 字体包 `FZSSK.ttf`。
2. 大章节标题：黑体三号；缺失时用 `HYZhongJianHeiJ.ttf` 或 `HYQiHei-55J.ttf`。
3. 正文：仿宋_GB2312 三号；缺失时用 `FZFSK.ttf`。
4. 正文小标题：楷体_GB2312 三号；缺失时用 `HYKaiTiJ.ttf`。
5. 表格表头浅蓝底色；行高舒展，长文本列加宽。
6. PDF 段落启用 CJK 换行，尽量使用 `ERP使用`、`ERP信号` 等无空格写法，避免中英文硬断行。
7. 标题不附加日期；日期只体现在文件名。

## 十六、0 人和缺失数据总规则

1. 任何章节统计为 0 人时，只写 `当前未发现` 或 `暂不涉及`，不得继续写 `其中`、`主要包括`、`建议名单包括`。
2. 某输入文件缺失时，应尽量生成可用章节，并在相应章节写明缺失，不得让脚本报错中断。
3. 空表可以保留表头，但综合报告正文中应优先用文字说明，不展示大量空表。
4. 主管评价样本为 0 时，综合结论第2、3条应写 `当前未取得主管评价样本，暂无法形成原始主管评价级别/20题规则重算级别分布`，不得写 `无` 后继续比较。

## 十七、最终验收清单

1. 所有分报告和综合报告均有 MD 和 PDF。
2. 文件名日期为运行当天日期，正文标题不带日期。
3. ERP 口径为 `2025年全年`。
4. `主管评级对应公司等级` 未出现在分析判断中。
5. 妈妈岗名单不含无意向人员。
6. 证书报告中 HR 复核有证书人员已从无证书名单移除。
7. 综合报告中没有 `当前0名`、`无。其中`、`0人主要意见包括` 等矛盾表述。
8. 单部门/部分部门/全部部门的范围称谓一致。
9. 宽表列宽合理，长文本不竖排，表头有浅蓝底色。
10. 主管评价样本为 0 的部门可以完整生成报告，不报错。
