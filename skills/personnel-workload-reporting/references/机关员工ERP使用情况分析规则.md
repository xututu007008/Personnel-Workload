# 机关员工ERP使用情况分析规则

## 一、规则定位

本文件用于指导生成、检查和修改 `机关员工ERP使用情况分析`。本报告从 ERP 流程发起、审批参与、评论、驳回、处理时长等维度观察员工数字化使用痕迹，是工作量摸排综合判断的辅助维度。

## 二、必须读取的输入

1. `ERP使用情况.xlsx`
   - `2025年全年` 工作簿：发起流程、参与流程、审核小时、评论次数、驳回次数等。
   - `职能部门人员台账` 工作簿：机关员工范围、部门、是否中层、优秀情况、安全先进个人情况。
2. `职能部门工作量摸排表*.xlsx`
   - 限定当前部门范围，补充员工名单。
3. `部门成员工作主管评定*.xlsx`
   - 限定当前评价范围，读取主管主观评级。
4. `机关员工主管评价20题重算报告`
   - 获取 `按20题计算等级`。

## 三、统计范围

1. 人员范围必须参考 `ERP使用情况.xlsx` 中 `职能部门人员台账`，不能只看 ERP 操作表。
2. 剔除公司领导、待岗人员、中层干部等不纳入机关员工统计范围的人员。
3. 当前文件夹如只放一个或多个部门数据，应只统计当前范围人员。
4. ERP `2025年全年` 源表中出现但不属于当前范围的人员，不纳入本报告人数统计。
5. 同一员工存在多条 ERP 记录时，按姓名汇总并归属到当前台账部门。

## 四、核心指标

| 指标 | 计算说明 |
| --- | --- |
| 纳入统计员工 | 当前范围有效员工数 |
| 有 ERP 操作痕迹 | 发起流程数或参与流程数大于 0 |
| 无 ERP 操作痕迹 | 发起流程数和参与流程数均为 0 |
| 发起流程总数 | 当前范围人员发起流程数合计 |
| 参与流程总数 | 当前范围人员审批/参与流程数合计 |
| 审核总小时 | 当前范围人员审核耗时合计 |
| 平均审核小时 | 审核总小时 / 参与流程总数 |
| 评论率 | 有效评论次数 / 参与流程总数 |
| 驳回率 | 驳回次数 / 参与流程总数 |

分母为 0 时显示 `0` 或 `-`，不得报错。

## 五、角色分类

| 分类 | 判定 |
| --- | --- |
| 兼有发起和审批 | 发起流程数 > 0 且参与流程数 > 0 |
| 仅发起 | 发起流程数 > 0 且参与流程数 = 0 |
| 仅审批 | 发起流程数 = 0 且参与流程数 > 0 |
| 无ERP痕迹 | 发起流程数 = 0 且参与流程数 = 0 |

## 六、必须包含的章节

1. 统计范围
2. 整体结果
3. 发起流程与参与流程概况
4. 部门分布
5. 角色特征分布
6. 按人员评价等级观察ERP使用情况
7. 重点人员表现
8. 无ERP痕迹情况
9. 异常信号
10. 结论判断

其中 `重点人员表现`、`异常信号`、`结论判断` 为必保章节，不得因当前范围人数少而删除。

## 七、重点人员表现规则

至少包含：

- 发起流程数量靠前人员
- 参与流程数量靠前人员
- 评论率较高人员
- 发起流程较少人员
- 参与流程较少人员

人数不足 10 人时按实际人数列示。

## 八、异常信号规则

至少观察：

1. 高流程量但评论率偏低。
2. 高流程量且平均处理时长偏高。
3. 参与流程平均审核时间超过阈值。
4. 参与流程存在但无任何审批意见和驳回行为。
5. 主要人员与异常情况交叉。

阈值应在脚本中集中定义，修改时不要散落在多个章节。

## 九、表格与 PDF

1. 表头浅蓝底色。
2. `发起流程概况`、`角色特征分布` 等少列表应等距或近似等距。
3. 部门分布宽表应压缩数字列，保留部门列和比例列可读。
4. 行高适中，不得贴线。

## 十、验收清单

- 机关总人数与 `职能部门人员台账` 一致。
- 无 ERP 痕迹人员没有遗漏。
- 当前范围外 ERP 人员没有混入。
- 等级字段来自 20 题重算报告。
- 综合报告引用的 ERP 信号与本报告一致。

## 十一、逐项输出标准

| 章节 | 必须统计的项目 | 标准 |
| --- | --- | --- |
| 统计范围 | 当前范围、部门、人数、来源 | 必须说明人员名单来自台账、工作摸排和主管评价的交叉范围。 |
| 整体结果 | 纳入统计员工、有痕迹、无痕迹、源表流程人数 | 必须同时说明 ERP 源表原始人数和本报告纳入人数，避免误解。 |
| 发起流程概况 | 涉及人数、发起流程总数 | 发起人数为发起流程数大于 0 的人数。 |
| 参与流程概况 | 涉及人数、参与流程总数、审核小时、评论率、驳回率 | 评论率和驳回率以参与流程总数为分母。 |
| 部门分布 | 部门人数、有痕迹、无痕迹、发起、审批、流程量、评论率、驳回率 | 单部门也要保留部门分布表。 |
| 角色特征 | 兼有发起审批、仅发起、仅审批、无ERP痕迹 | 四类互斥，合计应等于当前范围人数。 |
| 等级观察 | 按 20 题重算等级分组 | 每组展示人员 ERP 指标和主管评价等级。 |
| 重点人员表现 | 发起靠前、参与靠前、评论率高、发起少、参与少 | 人数不足 10 人时按实际人数。 |
| 异常信号 | 低评论率、高处理时长、无评论驳回、高流程交叉 | 0 人时写无，不删除小节。 |
| 结论判断 | 总体参与、异常范围、管理含义 | 结论必须与异常信号名单一致。 |

## 十二、人员级字段标准

人员明细或等级观察表中，字段口径如下：

| 字段 | 说明 |
| --- | --- |
| 部门 | 优先取职能部门人员台账部门；当前范围收紧时按当前部门显示。 |
| 姓名 | 统一清理空格和换行。 |
| 发起流程数 | `2025年全年` 发起流程合计。 |
| 审批流程数 | `2025年全年` 参与/审批流程合计。 |
| 审批驳回率 | 驳回次数 / 审批流程数。 |
| 审批评论率 | 有效评论次数 / 审批流程数。 |
| 审批平均处理小时 | 审核总小时 / 审批流程数。 |
| 按20题计算等级 | 来自主管评价重算报告。 |
| 主管评价等级 | 主管主观评级，仅作对照。 |

## 十三、可执行生成流程

1. 打开 `ERP使用情况.xlsx`，必须读取 `2025年全年` 和 `职能部门人员台账` 两个工作簿。
2. 清洗字段：`部门`、`人员名称`、`参与类型` 去空格和换行；`参与流程数量`、`处理时间（分钟）`、`驳回次数`、`有效评论次数` 转为数值，空值按 0。
3. 按 `人员名称 + 参与类型` 汇总流程数量，形成 `发起者` 和 `参与审批者` 两列；缺失参与类型列时补 0。
4. 按 `人员名称` 汇总处理时间、驳回次数、有效评论次数。
5. 构造当前范围人员底表：优先使用当前工作摸排表和主管评价表中出现的人员；若是全机关报告，则使用 `职能部门人员台账` 并按机关口径剔除公司领导、待岗、中层。
6. 将 ERP 汇总结果按姓名左连接到人员底表。没有 ERP 记录的人员所有数值列补 0。
7. 根据主管评价重算报告补入 `按20题计算等级`，根据主管评价表补入 `主管评价等级`；主管评价样本为 0 时等级写 `未评级` 或 `暂无评级`。
8. 计算人员级指标：流程合计、审批驳回率、审批评论率、审批平均处理小时、总评论率、平均处理小时。
9. 生成部门汇总、角色特征、等级观察、重点人员表现、无痕迹名单、异常信号和结论。
10. 输出 MD 后生成 PDF，表格表头浅蓝，少列表等距，宽表按内容调整列宽。

## 十四、阈值与异常定义

| 异常项 | 默认阈值 | 输出要求 |
| --- | --- | --- |
| 高流程量低评论率 | 流程合计 >= 500 且总评论率 < 2% | 列部门、姓名、流程合计、有效评论次数、评论率。 |
| 高流程量高处理时长 | 流程合计 >= 100 且平均处理小时 >= 80 | 列部门、姓名、流程合计、平均处理小时。 |
| 审批平均处理超长 | 参与审批者 > 0 且审批平均处理小时 >= 120 | 单独列 B 及以上或重点等级人员。 |
| 有审批无意见驳回 | 参与审批者 > 0 且有效评论次数 = 0 且驳回次数 = 0 | 说明可能是低风险流程或留痕不足，不直接负面判断。 |
| ERP痕迹较少 | 流程合计 <= 52 | 与重算等级交叉，D及以上人员进入综合报告辅助复核。 |

阈值应集中维护；如用户要求调整阈值，所有章节和综合报告引用需同步更新。

## 十五、空数据与边界场景

1. 当前范围人员为 0 时，报告仍生成统计范围和整体结果，人数写 0，其余章节写 `当前无可统计人员`。
2. 某类重点人员或异常信号为 0 时，保留小节并写 `无`，不得套用“主要包括”等固定句。
3. `流程合计 >= 100` 的人员为空时，评论率排序不得使用会导致空表赋值报错的行级 `apply`；应使用向量化计算或先判断空表。
4. `2025年全年` 工作簿缺失时不得自动回退 `近三年`，应提示输入文件口径不满足当前规则。
5. 人员存在于 ERP 操作表但不在当前范围底表时，不纳入人数统计，可在需要时作为源表外人员说明。

## 十六、结论写法

1. 结论必须围绕当前范围，不得在单部门报告中写“机关整体”。
2. ERP 只作为工作承担度和流程留痕辅助信号，不得直接决定人员等级。
3. 流程量高、驳回多、评论多一般作为承担度或把关责任信号；只有结合岗位职责和主管评价后才可形成复核建议。
4. 无 ERP 痕迹不等于无工作量，应结合岗位是否使用 ERP、工作摸排、主管评价判断。

## 十七、生成后复核方法

1. 核对 `有ERP操作痕迹人数 + 无ERP操作痕迹人数 = 纳入统计员工`。
2. 核对四类角色特征人数合计等于纳入统计员工。
3. 随机抽查 3 名人员，从源表汇总其发起和审批流程，确认与报告一致。
4. 检查综合报告 ERP 信号名单与本报告重点人员和异常信号一致。
